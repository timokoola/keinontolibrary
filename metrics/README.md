# Metrics — system-test accuracy

A reproducible pipeline that measures how well the declension engine reproduces the
reference corpus, and renders a shareable graphic.

```
Kotus word list ──▶ every noun × (15 cases × 2 numbers) ──▶ generated by the rule engine
                                                              │
                          reference corpus (generated  ◀──────┘ compared per slot
                          with Voikko) = ground truth
                                                              ▼
                                           success rates by case / number / class
                                                              ▼
                                                chart.py ──▶ PNG for sharing
```

## Regenerate

```sh
# 1. produce the packed artifact (needs the local source data — see ../LICENSING.md)
cargo run --release -p keinontolibrary-ingest --bin keinontolibrary-ingest

# 2. compute metrics -> metrics/metrics.json
KEINONTO_METRICS=metrics/metrics.json \
  cargo run --release -p keinontolibrary-ingest --bin keinontolibrary-metrics

# 3. render the graphic -> metrics/keinontolibrary-accuracy.png
python3 -m venv .venv && .venv/bin/pip install matplotlib
.venv/bin/python metrics/chart.py
```

## Files

- `metrics.json` — raw success counts bucketed by `(case, number)` and by declension class.
- `chart.py` — renders the LinkedIn graphic from `metrics.json`.
- `keinontolibrary-accuracy.png` — the shareable graphic.
- `linkedin-post.md` — draft post.

## What the numbers mean

- **Coverage** = slots where the engine produces *some* form ÷ reference slots.
- **Agreement** = slots where the engine's output contains the reference's primary form ÷
  slots the engine produced. This is the headline accuracy.

The reference corpus is *our* form set (generated with the Voikko tool), so this measures
the engine against an independent morphological generator — not against itself.
